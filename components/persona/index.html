Components Persona

The persona control is used to represent a person. It comes in different sizes and can optionally show the presence state of the user.

Back to top

Usage

The persona control comes in varrying sizes and is comprised of a handful of HTML elements. Everything is wrapped in a single <div> with the class ms-Persona. By default the medium size is used, but you can control the size of the control with an additional class indicating the size:

  • Tiny: .ms-Persona--tiny
  • X-Small: .ms-Persona--xs
  • Small: .ms-Persona--sm
  • Large: .ms-Persona--lg
  • X-Large: .ms-Persona--xl

Within this container there are two sub-containers - one <div> for the image (.ms-Persona-imageArea) and one <div> for the details pane (.ms-Persona-details).

The smaller controls that don't use an image don't need to include an image, but they can include a <div> for the presence details (.ms-Persona-presence).

The details container can contain multiple levels of text depending on the size of the control. The levels of text should be prioritized using the following classes:

  • .ms-Persona--primaryText
  • .ms-Persona--secondaryText
  • .ms-Persona--tertiaryText
  • .ms-Persona--optionalText

The persona control loads a default placeholder image while the referenced person's image is loading or if it fails to load so a broken image is not shown.

Darker Text

In cases when the control is placed on a darker background color and you need more contrast with the text, you can use the class .ms-Persona--darkText in the outer container to darken the text up.

See the Pen Office UI Fabric - Persona (DarkerText) by Andrew Connell (@andrewconnell) on CodePen.

Selectable

Typically the controls are for presentation only, but if you add a .ms-Persona--selectable the control will change the cursor to a pointer indicating the user can select it.
