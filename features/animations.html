Features Animations

The Office UI Fabric includes common animations that you will find used across Office 365, Office clients & SharePoint. These animations are provided as CSS classes that you can use within your own applications.

The animations vary in three different axis:

  • Direction: Aside from the common up / down / right / left direction options, animations can also move related to the origin and completion of tasks or occurance of events.
  • Orientation: How does the animation behave? Animations in the Office UI Fabric can slide in/out , scale or fade in/out, or zoom in/out.
  • Speed / Duration: How long does it take for the animation to complete. The speed of an animation conveys the importance or urgency of the action or subject of the animation.

Usage

Include a reference to the fabric.css file, or the corresponding minified version fabric.min.css. These are available from the public CDN:
<link rel="stylesheet" href="//appsforoffice.microsoft.com/fabric/1.0/fabric.min.css" />

Using any of the animations provided is straightforward:

  • Create an element that you want to animate and position it in the final position.
  • If the element is going from not visiable to visable, such as fading in, set it's opacity to zero. The animation will handle the trasition to 1.
  • Conversely, if the element is going from visiable to not visable, such as fading out, set it's opacity to 1. The animation will handle the trasition to 0.
  • To apply the animation, when the desired event occurs that should trigger the animation, add an animation class to the element.
  • After adding the animation, it's a good idea to set a timeout to remove the animation after it completes (500ms or 1s works for this) by removing the class you added.

Below you will find examples of all the animations in the Office UI Fabric. Let's look at a simple example first. Jump down to the HTML part of the sample. What you will see is a div that contains two other div elements. The outer one is the container for the object (.animation-content) and the animated object (.animate-panel).

When the .animation-content is clicked, the script adds the animation class .ms-u.slideRightIn40 to the .animation-panel and sets a 1 second timeout to remove the class which happens well after the completion of the animation.

<div class="ms-font-su">.ms-u-slideRightIn40</div>
      
      <div class="animation-container">
      <div class="animation-content" onclick="triggerAnimation(this);">
        <hr /><hr /><hr /><hr /><hr /><hr /><hr /><hr />
      </div>
      <div class="animation-panel ms-bgColor-themePrimary"></div>
      </div>
      
      <div class="ms-font-l">Click anywhere in the box above to trigger the animation.</div>

Panel Animation Guidance

Microsoft offers the following guidance when applying animations to panels:

When choosing a motion for side panels, consider the origin of the triggering element. Use the motion to create a link between the action and the resulting UI. For example, if the triggering element is on the right side of the interface, consider having the panel move in from the right.

Dialogs Animation Guidance

Microsoft offers the following guidance when applying animations to dialogs:

When choosing a motion for dialogs, consider the origin and tone of the content. For a warning or error dialog, a quick fade in might be appropriate. If the dialog appears when a user chooses an item to get more information, a scale-up might be appropriate.
