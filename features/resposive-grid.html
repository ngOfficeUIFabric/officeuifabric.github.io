Features Responsive Grid

Like other popular design libraries, the Office UI Fabric includes a mobile and fluid responsive grid system that is based on the popular 12 column layout. Use any combination of columns with provided CSS classes that add up to 12 columns for a strong layout.

Back to top

Usage

Include a reference to the fabric.css file, or the corresponding minified version fabric.min.css. These are available from the public CDN:
<link rel="stylesheet" href="//appsforoffice.microsoft.com/fabric/1.0/fabric.min.css" />

Let's look at how the grid works. It is all based on a container and then a series of rows and columns:

  • Rows must be placed inside a grid using .ms-Grid.
  • Columns are then placed within rows using .ms-Grid-row.
  • Nothing else is placed within the rows; only columns should be placed in rows.
  • Columns are created using .ms-Grid-col followed by a sizing class.
  • Size the columns using classes in the format of .ms-u-sm[#]. For example, the very first row in the first grid above contained 12 <class="ms-Grid-col ms-u-sm1"> elements.
  • The small sizing class must be used .ms-u-sm[#], but the medium and large sizes are optional.
  • The column classes must add up to 12 for each row using any combination of sizing classes.
  • Optionally add additional sizing classes for bigger screens. These are in the format of .ms-u-md[#] and .ms-u-lg[#].
